# 稳定性完善设计

## 目标

针对本轮 Code Review 暴露的稳定性问题，完善题库持久化、题库编辑、AI 异步回调和自动循环调度。保持现有业务行为不变，不处理需要真机节点验证的考试及冷门题型。

## 设计

### 题库原子写入

所有 JSON 写入先写入同目录临时文件，成功关闭后再替换目标文件。目标文件存在时使用备份/替换策略，写入失败保留原文件。读取损坏文件时继续返回空结果，但保留损坏文件供排查。

### 题库编辑 key 一致性

编辑入口同时归一化旧题干和新题干。删除旧项、写入新项均使用归一化 key，避免标点和空白差异造成重复条目。

### AI 异步请求隔离

为每个题干维护请求代次。异步结果回调携带代次和题干规范 key，只有仍是当前请求且调用方仍关注该题时才回调 UI。缓存和去重行为保持不变。

### 调度器取消

`WaitForScheduler` 增加取消 API，移除已排队的 Handler 回调，并在服务停止/清理时调用。现有 `active()` 检查保留作为第二层保护。

## 测试策略

- 增加纯函数/可测试辅助逻辑的单元测试。
- 验证题库编辑对旧题干归一化后不会残留旧 key。
- 验证异步请求只接受当前代次结果。
- 验证构建与现有全部单测。
- 不声称已完成真机行为验证。

## 非目标

- 不迁移个人发布签名配置。
- 不重写题库格式。
- 不改变 AI 服务商、请求协议或 UI。
- 不实现新的题型解析。

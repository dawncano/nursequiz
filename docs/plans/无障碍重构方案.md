# 无障碍节点重构方案

> 目标：把"读题/点击"底层从【截图+OCR+几何解析】换成【无障碍节点树】，并把题库改成
> "分库 · 题干→正确答案文字"。一步一步来，本次只做下面圈定的范围。

## 一、本次范围

**做：**
1. **读题底层**：截图+OCR → 无障碍节点树（彻底告别 OCR / 识别错误 / 替换表）。
2. **数据层**：分库匹配 + 存"题干（归一化）→ 正确答案**文字内容**"。
3. **悬浮答案手动模式**：悬浮窗显示正确答案文字，人工自己点（不自动点，更隐蔽）。
4. **AI 未命中钩子**：预留接口，**本次空实现**（留好"未命中→AI"那条路，先不接）。

**不做（后续单独迭代）：** AI 实接 · 云端众包热更 · Shizuku 自动授权。

---

## 二、读题底层：OCR → 节点树

- **删**：`OcrEngine` / `OcrFix` / `OcrLearn` / 读题用的 `captureScreen`。
- **改**：`ScreenParser` → `NodeParser`，遍历当前窗口 `AccessibilityNodeInfo`，产出 `ScreenModel`。
  `act()` 主循环**基本不动**（NodeParser 出同结构 ScreenModel = 换发动机不换车架）。
- **解析规则**（基于实测 `apk_analysis/captures/current/hlzs_q.xml` / `apk_analysis/captures/current/hlzs_m.xml`）：
  - **题型**：节点 text 含 单选题/多选题/判断题/填空题。
  - **题干**：题型节点后、首个选项前的长文本 → 归一化 `replace(/[^一-龥\d]/g,"")` 只留中文+数字。
  - **选项**：`clickable=true` 且接近整行宽的 ViewGroup，按 `bounds.top` 排序 = A→E（实测一个不漏）；每项文字 = 其子树里去掉字母后的文本。
  - **正确答案**（反馈态）：text 含"正确答案：X"，X 是**字母** → 需结合当前选项文字映射成文字（见数据层）。
  - **按钮**：确定 / 下一题 / 上一题。
  - **点击**：`node.performAction(ACTION_CLICK)` 优先，失败回退点 bounds 中心。
  - **时序**：用 `waitFor("纠错上报"节点出现)` 替代 sleep 硬等，确保页面就绪再读/点。

---

## 三、数据层重构

- **现状**：`AnswerStore` 已按 "标题+项目" 分库；每库存 题干→字母（如 `ACD`）。
- **改为**：每库存 **题干（归一化）→ 正确答案文字**。
  - 单选：正确选项文字（如 `药物降温`）
  - 多选：多个选项文字，分隔符连接（如 `选项1|选项2`）
  - 判断：`对` / `错`
  - 填空：答案文字
- **分库匹配**：沿用 `setBank(标题, 项目)` 分库；库内按归一化题干匹配（归一化后精确匹配即稳，可选 LIKE 容错）。
- **入库（学习）**：反馈页读"正确答案：字母" + 当前选项文字列表 → 字母映射到选项文字 → 存文字
  （参考竞品 `findAnswer`：字母 `D` → 选项[3] 文字）。

---

## 四、作答流程

```
节点读题 → 归一化题干 → 查当前分库
 ├─ 命中（拿到答案文字）
 │    ├─ 自动模式：在选项节点里找文字匹配的 → performAction点击 → 点确定 → 等下一题
 │    └─ 悬浮模式：把答案文字显示在悬浮窗，人工自己点（不自动点；无答案显示空/undefined）
 └─ 未命中
      ├─ AiHook.resolve(题干, 选项) —— 本次空实现，返回 null
      └─ 兜底：盲选学习（随机点→提交→反馈页读正确答案→映射文字→入库；下次命中）
```

> 存"文字"的好处：① 抗选项乱序（靠 `text` 匹配选项，不靠 A/B/C 下标）；② 悬浮模式直接显示；
> ③ 自动模式点击也用同一份文字定位。三处统一。

## 五、模块改动清单

| | 模块 |
|---|---|
| ❌ 删 | `OcrEngine`、`OcrFix`、`OcrLearn`、读题 `captureScreen` |
| 🔧 改 | `ScreenParser`→`NodeParser`；`ScreenModel`：`options` 改为 `List<Option(text, node/bounds)>`，新增 `correctAnswerText`；`AnswerStore`：值 字母→文字、`keyFor` 归一化、保留分库；`act()`：命中点击改"按文字找节点点"、新增悬浮模式分支；`FloatingOverlay`：加悬浮答案显示窗 + 模式开关 |
| ➕ 新增 | `NodeParser`；`AiHook`（空接口：`fun resolve(q, options): String? = null`） |

---

## 六、落地步骤（分阶段，每步可真机验）

1. **探针**：`NodeParser` 解析一道题（题型/题干/选项文字/答案），广播触发、存文件，真机 dump 验证解析对不对。
2. **点击**：接 `performAction` 点选项 + 确定，跑通"自动答一道单选"。
3. **数据层**：`AnswerStore` 改存文字 + 归一化 key；反馈页读答案映射文字入库。
4. **切主循环**：`act()` 全走节点，删 OCR 三件套；跑通整组自动答题。
5. **悬浮答案模式**：加显示窗 + 模式开关（自动/悬浮 可切）。
6. （后续迭代）AI 实接 / 众包 / Shizuku。

---

## 七、实现中待验证

- **选项乱序**：dump 同题两次对比顺序（存文字已天然抗乱序，主要确认点击稳定）。
- **performAction 是否生效**：不生效则回退点 bounds 中心。
- **多选/判断/填空节点结构**：切到这些题型各 dump 一次确认。

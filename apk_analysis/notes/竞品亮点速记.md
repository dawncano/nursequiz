# 竞品「小助手」值得学习的点（速记）

> 来源：AutoJs Pro 套壳外挂，目标=护理助手(RN)。逆向自 main.js.bak 等明文备份。

## 1. 控件树读题（替代 OCR）⭐核心
- `className("android.widget.TextView").text(x).findOnce().parent().click()`
- 直接读无障碍节点文字、点控件容器，**零识别错误、不漏选项、不怕 FLAG_SECURE**。
- 已实测：护理助手单选/多选题干+选项+正确答案全可读 → 我们也走这条。

## 2. 自动开启无障碍服务（免去手动设置）
- 三路：`root` / `Shizuku(rikka.shizuku，免root借ADB权限)` / 直接写 `secure settings`。
- 对应函数：`enableAccessibilityServiceByRoot` / `BySecureSettings`。
- 价值：装完一键开服务，不用每次进系统设置点。重装后尤其省事。

## 3. 题库 miss → 实时问 AI（新题也答对）
- 本地题库查不到 → 调云端：通义千问(`luckycola/aliai/tyqw`) 故障转 GPT(`openaiV3`)。
- Prompt 极简：「回答以下问题，仅提供答案对应ABC及中文即可，不用额外解释。」+题目。
- 发的是控件读到的纯文本 → 用文本 LLM 即可，不需要视觉 VLM。

## 4. 题库热更 + 众包
- `下载题库.js/updateTiku.js` 服务器拉取增量更新本地 tiku.db(SQLite)。
- `上传题库.js` 把新遇到的题自动上传汇总 → 全体用户共享。
- 解决「每月换新题」：一处更新，所有设备热更。

## 5. 视频自动挂课
- 读「累计时长-已学时长」算需挂多久，挂机 sleep；每15s防息屏、每3s点播放防暂停。
- 凑满学时考核。和答题是独立功能。

## 6. 伪装 / 重打包（技术存档，自用不建议照搬）
- 无障碍服务名冒充 `com.google.android.accessibility.selecttospeak.SelectToSpeakService`（装成 Google 系统服务，降警惕/避按名检测）。**新版 9.3.11 真机实测仍是这个伪装名，未变**（`dumpsys package` 确认）。
- 目标 app 是 `SignatureKiller843` 重打包破解版（绕签名校验）——与自动答题无关，多为解锁VIP/去广告。自己的产品光明正大即可，这条不必学。新版公告里要求「护理助手破解版 6.7.9」（旧版速记记的是 7.6.9，按版本演进对待即可）。

## 7. 源码里的工程细节（读 main.js.bak / updateTiku.js.bak 挖到）
- **题干归一化** `question.replace(/[^一-龥\d]/g,"")`：只留中文+数字，标点/空格/字母全删 → 题库 key 天然稳定，不管读到的标点空格怎么抖都命中同一条。**正好治我们 key 漂移的老毛病。**
- **模糊匹配 + 防注入**：`WHERE question LIKE ?` 参数化查询 + 转义 `%_`（`replace(/[%_]/g,"\\$&")`）→ 题目有细微差异也能命中，且防 SQL 注入。比精确 key 匹配鲁棒。
- **多级兜底读题**：先递归收集节点文本 → 正则提题干 → 失败再用不同 `depth(23/24)` 控件兜底。适配护理助手多个版本的控件层级差异。
- **考试/练习双路径**：`textStartsWith("考生")` 区分两种界面，各走各的节点解析。
- **随机延时** `randomDelay(min,max)`：模拟人操作节奏，避免机械固定间隔被风控识别。
- **资源/健壮性**：DB 查询 try/catch/finally 关游标+连接、空值校验、child 有效性检查。
- **演进痕迹**：注释「不截图了，答错后查找正确答案」「对应护理助手6.2.0」——它早年也走过截图，后来转控件树。**和我们现在要走的路一模一样，它只是早走通了。**

## 8. 运行保护 & 答题健壮性（续读挖到）
- **waitFor 等控件就绪**：多处 `text("纠错上报").waitFor()` / `id("button1").waitFor()` 确保页面加载完再读/点。**治时序——比我们截图方案里 sleep 硬等优雅。**
- **脏数据自愈**：检测到错误答案格式（`","` / `"&,&,"`）→ 自动 `DELETE` 清掉题库脏记录 + 震动提示。题库自我修复。
- **图片题检测**：`child(4).clickable()` 判断是否图片题 → 延长 2s 等图加载、特殊点击。
- **盲选建库**：无答案时 `randomClick` 随机点 → 提交 → 读「正确答案」节点 → 入库。和我们盲选学习同思路，但它读答案是节点直读、零误差。
- **滚动读长题**：题目可滚动则 `scrollDown` 读全。
- **选项几何对齐提取**：按选项字母左边界 x 对齐筛同列 TextView 取选项文字，并处理「点击前/后」控件 depth 变化（21/22）。
- **题型全覆盖**：单选/多选/判断走选项，填空/简答 `setText("。。。")` 占位提交；各自处理「下一题/提交/所有题目已作答」边界。
- **禁返回退出 + 音量键退出**：拦 `back_pressed` 防误触退出，音量+键作退出快捷键（子线程 observeKey）。
- **全局日志落盘** `/sdcard/log.txt`，便于无人值守排错。
- **悬浮窗权限 3 种申请法**+降级，带各版本兼容注释。
- **瑞科授权心跳**：心跳失败回调 → 强制关 app（商业防破解/防共享卡密标准做法）。

## 9. 最后一批（UI / 主循环 / 视频，已读到底）
- **两种使用形态**（产品思路值得想）：`win2`「悬浮答案」只在小窗显示答案文字、人工自己点——不自动点、更隐蔽、像学习辅助；`win`「自动点击」点一下答一题；`main3` 自动刷视频。
- **答错自动修正题库**：点完比对「正确答案 vs 你的答案」，不一致就 `UPDATE tiku` 改答案。题库自愈进阶。
- **两级题库**：本地 `tiku.db` 查不到 → 再查共享 `下载题库.txt`（`findtextAnswer` 用 `Map` 一次性解析+查，O(1)）。
- **众包阈值**：新增满 100 题自动触发 `上传题库.js`。
- **连续答对计数**：`p>4200` 认为题库刷全了，自动 `exit()`。
- **防风控**：每题间 `randomDelay(2000,2500)`。
- **视频**：`shiping` 遍历任务列表 → `read` 按「累计-已学」挂时长 → `main3` 调度 + 独立线程监控「播放完毕是否播放下一个」自动点取消。
- **明确依赖破解版**：`launchApp("护理助手'")` 失败弹「请确认为最新破解版」——印证它绑定破解版（多为解锁题库/按名定位），**与自动答题技术能力无关**（原版无障碍照样可读，已实测）。
- **工程卫生**：DB 操作全走 `threads.start` 子线程避免卡 UI；题干增删改一律先归一化。

## 10. AI 调用接口（题库 miss 时的新题兜底）
**机制**：控件读到的纯文本题目 + 一句极简 prompt → HTTP POST → 取 `data.result`。
```js
http.post("https://luckycola.com.cn/aliai/tyqw", {
    ques: "回答以下问题，仅提供答案对应ABC及中文即可，不用额外解释。" + question,
    appKey: '6762c46838ccf94f638c8806',
    uid:    '8LGaTT1734526010879DqGSAetQlM',
    isLongChat: 0          // 单轮、不带上下文
}, { headers: { "User-Agent": "okhttp/4.9.1" } });
// 返回 JSON.parse(body).data.result 即答案
```
- **调的 AI**：阿里通义千问(`/aliai/tyqw`，主力) + OpenAI GPT(`/openai/openaiV3`，通义故障自动转)。
- **prompt 按题型微调**：选择题「只给ABC+中文」/ 填空「只给中文」/ GPT 版「只给序号+文字」。
- **免费还是计费**：**没直连官方**，走第三方聚合平台 `LuckyCola(luckycola.com.cn)` 中转，`appKey/uid` 是平台账号、带免费额度/积分 → 作者基本白嫖。代价：有限额、依赖别人平台稳定性、`appKey` 明文写死在脚本里（谁都能蹭）。

**我们仿制怎么做（更稳）**：
- 别用第三方中转，直接用**阿里通义千问官方 DashScope API**（新用户免费 token，超出计费极便宜、国内直连无墙）；
- prompt 直接抄它的（已验证好用）；
- 只在**题库 miss 的新题**才调 AI → 答对入库 → 之后查表零成本。

## 11. 新版 9.3.11 真机实测验证（本轮 2026-06-24，截图存 `apk_analysis/captures/flow_capture/`）

> 拿到的 `test.apk` = 「小助手。」**Pro 9.3.11-0 / 内部版本 2.0.6 / build 39**。和旧版最大差别：**`main.js` 已加密**（`project.json` `encryptLevel:1`，不再是明文），逆向变难——但本轮改用「装机实跑 + adb 抓状态」把全功能跑了一遍，源码层结论（1~10 节）基本都对得上。

### 启动 / 授权流程（实测）
- `SplashActivity` 启动即申请悬浮窗 → 跳系统「显示在其他应用的上层」→ 授权后回到登录。
- 登录页：卡密输入 + **记住卡密**（勾上 → 冷启动自动鉴权，无需手输）。实测卡密 `c76cf7dc75365665`，到期 `2026-07-01 08:43:32`。
- 启动公告：客服 QQ `2207186211`、售后群 `341141171`。
- **`force-stop` 会掉无障碍服务**，重启需重开（adb 直接 `settings put secure enabled_accessibility_services <伪装名>` + `accessibility_enabled 1` 可绕过手点）。
- 脚本控制：**音量上键停脚本**（实测 toast 提示，印证 8 节 observeKey）。

### 数据层（实测确认 SQLite 表结构）⭐补充
- 表名 **`tiku`**，字段 **`question` / `answer`**（CRUD 按钮直接把原始 SQL 当 toast 弹出来暴露的）：
  - 新增 → `INSERT INTO tiku (...) VALUES ('题','答')`
  - 修改 → `UPDATE tiku SET question=?, answer=? WHERE question=?`
  - 删除 → `DELETE FROM tiku WHERE question=?`（实测删一条，题库计数 138319→138318，确认真落库）
- 规模实测：**本地 138318 条**、**共享 1533 条**；共享题库启动自动拉 `下载题库.txt`（189KB）。

### 两种作答模式（实测，对应 9 节 win/win2）⭐
- **执行点击答题**（=`win` 自动点击）：启动「开启线程」，在屏幕**右中偏下角放触发热区**，触发即自动选中正确选项+点确定。早前一次实测**无人干预连刷 60 题**（计数 10→60，约 0.6s/题净速、含随机延时 2.2~2.3s）。
- **悬浮答案**（=`win2` 隐蔽手动）：屏幕**左中显示可拖动答案标签**，做题时标签从「。。。」变**答案原文**（实测某题显示 `<10%`，正是正确项），**不自动点击**、无答案则显示 `undefined`——人工自己点，更隐蔽。
- 注：adb 注入 `input tap` 点不动 AutoJs 的触摸热区（它收原始触摸事件），自动点击能力以那次 60 题连刷为准。

### 题库管理全功能（题库 tab，已逐个点过）
搜索（题目/答案 radio，SQLite LIKE，空值校验「请输入关键字」）· 最近十条 · 上一条/下一条（绑搜索结果集）· 重置 · 新增/修改/删除（见上）· **更新题库**（先弹「请确认护理助手为最新破解版」→ 进练习题抓取更新）· 刷新首页面（静默重扫护理助手首页）· 下载题库 · 上传日志/上传题库（静默上传）· 「更新题库时导出题库」勾选 · 自动刷视频（确认破解版 → 循环提示「请进入任务列表」挂课）。

### 对我们的直接启示
- 数据层就是一张极简 `tiku(question, answer)` 表 + 一个共享 txt 兜底——我们照此设计，零负担。
- **两模式分叉**（效率自动点 vs 隐蔽只提示）在真机上体验差异明显，是我们产品要明确的取舍点。
- 三个 CRUD 直接弹原始 SQL = 实现很糙，我们做正经的增删改 UI 就能甩开它。
